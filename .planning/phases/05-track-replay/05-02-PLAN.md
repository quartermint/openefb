<objective>
Build the replay UI (map, transcript panel, scrub bar, playback controls), flight history list, and wire everything into the Flights tab and FlightDetailView.

Purpose: Deliver the pilot-facing track replay experience -- the full visual loop of selecting a past flight, watching the route play back on the map with synchronized audio and scrolling transcript, and controlling playback via the scrub bar. This completes both REPLAY-01 and REPLAY-02 from the user's perspective.

Output: ReplayView (full-screen replay), ReplayMapView (replay-mode map), TranscriptPanelView, TimelineScrubBar, FlightHistoryListView, updated FlightDetailView with Replay button, updated ContentView Flights tab.

      - `var autoFollow: Bool = true` -- map auto-centers on marker; user pan disables, re-center button re-enables

      - `var isLoading: Bool = true`

Audio file resolution: `FileManager.default.urls(for: .applicationSupportDirectory, in: .userDomainMask).first!.appendingPathComponent(flightRecord.audioFileURL ?? "")`. Check FileManager.default.fileExists before passing URL. Sets isLoading = false. On error, sets loadError.
      - `func togglePlayPause()` -- calls replayEngine.play() or replayEngine.pause()
      - `func selectSpeed(_ speed: Float)` -- calls replayEngine.setSpeed(speed), hides speed picker
      - `func onUserPan()` -- sets autoFollow = false
      - `func recenter()` -- sets autoFollow = true
      - `func cleanup()` -- calls replayEngine.cleanup()
    - Available speeds: [1.0, 2.0, 4.0, 8.0] per user decision

    **2. ReplayView.swift** -- Full-screen SwiftUI view, the main replay experience:
    - Takes `flightRecord: SchemaV1.FlightRecord` as input
    - `@Environment(AppState.self) private var appState`
    - `@State private var viewModel = ReplayViewModel()`
    - Layout (iPad landscape optimized):
      - HStack:
        - Left: ReplayMapView (takes up remaining space, flex)
        - Right: TranscriptPanelView (~300pt wide, collapsible). When collapsed, show only current segment as floating card overlay on map. Toggle via button.
      - Bottom overlay (full width): TimelineScrubBar
      - Top-left overlay: Playback controls (play/pause button, speed selector, re-center button if autoFollow is off)
      - Top-right overlay: Instrument readouts from replay (GS, ALT, HDG in small format from replayEngine.currentSpeed, currentAltitude, currentHeading)
      - "Audio muted at [speed]x" indicator when replayEngine.audioMuted == true
    - onAppear: call viewModel.loadFlight(flightRecord:appState:)
    - onDisappear: call viewModel.cleanup()
    - Disable replay if appState.recordingStatus != .idle (per research pitfall 4). Show "Stop recording before replaying a flight." message.
    - Loading state: ProgressView while viewModel.isLoading
    - Error state: show viewModel.loadError with retry option
    - navigationTitle: "\(flightRecord.departureICAO ?? "") to \(flightRecord.arrivalICAO ?? "")"
    - navigationBarTitleDisplayMode: .inline

    **3. ReplayMapView.swift** -- UIViewRepresentable for the replay map:
    - Properties: `let mapService: MapService`, `let replayEngine: ReplayEngine`, `let autoFollow: Binding<Bool>`
    - makeUIView: Create a new MLNMapView instance (NOT reusing the navigation map instance -- per research recommendation, use a separate instance to avoid disrupting live navigation state)
    - Configure MapService with this map view
    - In Coordinator's mapView(_:didFinishLoading:) delegate callback:
      - Call mapService.addReplayLayers(to: style)
      - Call mapService.updateReplayTrack(replayEngine.trackPoints)
      - Call mapService.fitMapToTrack(replayEngine.trackPoints)
    - updateUIView: Called on engine state changes. On each update:
      - Call mapService.updateReplayMarker(coordinate: replayEngine.currentCoordinate, heading: replayEngine.currentHeading)
      - If autoFollow.wrappedValue: call mapService.centerOnCoordinate(replayEngine.currentCoordinate, animated: false) -- no animation at 20Hz
    - Detect user pan gestures via Coordinator implementing MLNMapViewDelegate regionDidChangeAnimated with reason .gesturePan -- set autoFollow to false
    - On disappear (via Coordinator deinit or explicit cleanup): call mapService.removeReplayLayers()

    IMPORTANT: ReplayMapView creates its OWN MapService instance internally (not sharing the navigation MapService from AppState). This prevents replay layers from polluting the navigation map.

    **4. TranscriptPanelView.swift** -- Scrolling transcript with highlight:
    - Takes `segments: [TranscriptSegmentRecord]`, `activeIndex: Int?`
    - ScrollViewReader wrapping ScrollView with LazyVStack
    - Each segment row shows: timestamp (formatted as mm:ss from audioStartTime), text, flight phase badge
    - Active segment highlighted with .blue.opacity(0.15) background and bold text
    - onChange(of: activeIndex): withAnimation(.easeInOut(duration: 0.3)) { proxy.scrollTo(index, anchor: .center) }
    - Empty state: "No transcript available" when segments is empty
    - Per user decision: transcript scrolls to matching segment as position marker moves

    **5. TimelineScrubBar.swift** -- Horizontal scrub bar with phase markers:
    - Takes: `currentPosition: Binding<TimeInterval>`, `totalDuration: TimeInterval`, `phaseMarkerFractions: [(phase: String, fraction: Double)]`, `isPlaying: Bool`, `onSeek: (TimeInterval) -> Void`
    - SwiftUI Slider(value:in:onEditingChanged:) bound to currentPosition, range 0...totalDuration
    - On editing changed (user dragging): call onSeek(newPosition) which calls replayEngine.seekTo
    - Phase marker overlay: ZStack with phase marker indicators positioned via GeometryReader at their fractional positions along the slider width. Each marker is a small vertical tick (2pt wide, 12pt tall) with phase name abbreviation below in .caption2 font.
    - Below slider: HStack with elapsed time (left, formatted mm:ss) and remaining time (right, formatted -mm:ss)
    - Per user decision: horizontal scrub bar at bottom with flight phase markers for visual navigation
    - Time formatting: helper function `formatTime(_ interval: TimeInterval) -> String` returns "HH:MM:SS" for >= 1 hour, "MM:SS" otherwise
