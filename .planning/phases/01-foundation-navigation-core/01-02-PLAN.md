<objective>
Build the aviation database layer: a GRDB-backed SQLite database with R-tree spatial indexes for airport proximity queries, FTS5 full-text search for airport/navaid lookup, and a bundled pre-built database with 20K+ FAA NASR airports. This is the data foundation that the map, weather, and navigation features query against.

Purpose: Provide instant offline access to all US airport data with sub-50ms spatial queries. The bundled SQLite approach (vs SwiftNASR on-device parsing) eliminates the 30-60 second first-launch penalty and works immediately offline.
Output: AviationDatabase actor, DatabaseManager service, and a bundled aviation.sqlite with 20K+ airports.
</objective>

    **AviationDatabase.swift** — `final class AviationDatabase: @unchecked Sendable`:

    Properties:
    - `private let dbPool: DatabasePool`

    Initialization (copy-on-first-launch pattern):
    ```swift
    nonisolated init() throws {
        let fileManager = FileManager.default
        let appSupport = fileManager.urls(for: .applicationSupportDirectory, in: .userDomainMask).first!
        let dbPath = appSupport.appendingPathComponent("aviation.sqlite")

        if !fileManager.fileExists(atPath: dbPath.path) {
            guard let bundledPath = Bundle.main.url(forResource: "aviation", withExtension: "sqlite") else {
                throw EFBError.databaseCorrupted
            }
            try fileManager.createDirectory(at: appSupport, withIntermediateDirectories: true)
            try fileManager.copyItem(at: bundledPath, to: dbPath)
        }

        var config = Configuration()
        config.prepareDatabase { db in
            try db.execute(sql: "PRAGMA journal_mode = WAL")
        }
        dbPool = try DatabasePool(path: dbPath.path, configuration: config)
    }
    ```

    Database schema (created by the importer tool, validated on open):
    - Table `airports`: columns icao TEXT PRIMARY KEY, faa_id TEXT, name TEXT, latitude REAL, longitude REAL, elevation REAL, type TEXT, ownership TEXT, ctaf_frequency REAL, unicom_frequency REAL, artcc_id TEXT, fss_id TEXT, magnetic_variation REAL, pattern_altitude INTEGER, fuel_types TEXT (JSON array), has_beacon_light INTEGER
    - Table `runways`: columns id TEXT, airport_icao TEXT REFERENCES airports(icao), length INTEGER, width INTEGER, surface TEXT, lighting TEXT, base_end_id TEXT, reciprocal_end_id TEXT, base_end_latitude REAL, base_end_longitude REAL, reciprocal_end_latitude REAL, reciprocal_end_longitude REAL
    - Table `frequencies`: columns id TEXT, airport_icao TEXT REFERENCES airports(icao), type TEXT, name TEXT, frequency REAL
    - Table `navaids`: columns id TEXT PRIMARY KEY, type TEXT, name TEXT, latitude REAL, longitude REAL, frequency REAL, elevation REAL, magnetic_variation REAL
    - Table `airspaces`: columns id TEXT PRIMARY KEY, name TEXT, class TEXT, floor_altitude INTEGER, ceiling_altitude INTEGER, center_latitude REAL, center_longitude REAL, radius_nm REAL, coordinates TEXT (JSON array of [lat, lon] pairs), min_lat REAL, max_lat REAL, min_lon REAL, max_lon REAL
    - Virtual table `airports_rtree`: USING rtree(rowid, min_lat, max_lat, min_lon, max_lon)
    - Virtual table `navaids_rtree`: USING rtree(rowid, min_lat, max_lat, min_lon, max_lon)
    - Virtual table `airspaces_rtree`: USING rtree(rowid, min_lat, max_lat, min_lon, max_lon)
    - Virtual table `airports_fts`: USING fts5(icao, name, faa_id, content=airports, content_rowid=rowid)

    Key query methods (all `nonisolated`):

    `func airport(byICAO icao: String) throws -> Airport?` — Single airport lookup by ICAO with JOIN to runways and frequencies tables. Build full Airport struct with nested Runway and Frequency arrays.

    `func airports(near center: CLLocationCoordinate2D, radiusNM: Double) throws -> [Airport]` — R-tree bounding box query: `latDelta = radiusNM / 60.0`, `lonDelta = radiusNM / (60.0 * cos(center.latitude * .pi / 180))`. Query airports_rtree for bounding box, then JOIN airports table. Return basic Airport structs (without runways/frequencies for performance — these are loaded on detail view).

    `func nearestAirports(to center: CLLocationCoordinate2D, count: Int) throws -> [Airport]` — Use `airports(near:radiusNM:)` with expanding radius (start 25 NM, double until count met, max 200 NM). Sort by great-circle distance using Haversine. Return top `count` results with runways included (for nearest airport emergency HUD).

    `func searchAirports(query: String, limit: Int = 20) throws -> [Airport]` — FTS5 query: `SELECT * FROM airports_fts WHERE airports_fts MATCH ? ORDER BY rank LIMIT ?`. Append `*` to query for prefix matching. Join with airports table for full data.

    `func airspaces(near center: CLLocationCoordinate2D, radiusNM: Double) throws -> [Airspace]` — R-tree bounding box query on airspaces_rtree. Return Airspace structs with parsed coordinates from JSON.

    `func navaids(near center: CLLocationCoordinate2D, radiusNM: Double) throws -> [Navaid]` — R-tree bounding box query on navaids_rtree.

    `func airspacesContaining(point: CLLocationCoordinate2D) throws -> [Airspace]` — For proximity alerts. First R-tree bounding box filter, then ray-casting point-in-polygon test for polygon airspaces, distance check for circular airspaces. Reference existing `pointInPolygon` implementation.

    3. Create R-tree virtual tables: `airports_rtree`, `navaids_rtree`, `airspaces_rtree`
    4. Create FTS5 virtual table: `airports_fts` with content sync to airports table

       - The top ~5,000 most-used public airports with basic data
       - Representative navaids (~2,000 VORs, VORTACs, NDBs)

    9. Print summary: total airports, runways, frequencies, navaids, airspaces inserted
