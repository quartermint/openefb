# Phase 1 -- UI Design Contract

> Visual and interaction contract for Foundation + Navigation Core. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform:** iPad, iOS 26, SwiftUI. This is a native app -- all values use Apple's design system conventions (Dynamic Type text styles, SwiftUI semantic colors, pt-based spacing). No web stack, no shadcn, no Tailwind.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | SwiftUI native (no component library) |
| Preset | not applicable (native iOS) |
| Component library | SwiftUI built-in + MapLibre `MLNMapView` UIViewRepresentable |
| Icon library | SF Symbols (system-provided, no third-party icons) |
| Font | System font (San Francisco) via SwiftUI `.font()` modifiers |

---

## Spacing Scale

Declared values in points (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 2pt | VStack spacing inside InstrumentCell (label-to-value gap) |
| sm | 4pt | Icon-to-text gaps within HStacks, divider horizontal padding |
| md | 8pt | Compact element spacing, badge internal padding vertical, bottom padding for instrument strip |
| lg | 12pt | Section spacing within sheets, controls VStack spacing, corner radii |
| xl | 16pt | Default horizontal padding for floating controls, map edge insets |
| 2xl | 20pt | Airport info sheet horizontal padding |
| 3xl | 44pt | Minimum touch target size for all interactive map controls |

Exceptions:
- 44pt minimum touch target on all map control buttons (Apple HIG mandate for iPad)
- 10pt internal padding on search bar and floating buttons (compact for always-visible controls)
- 6pt vertical padding on badges/chips (capsule form factor, tighter than standard)
- 40pt height for zoom +/- buttons within the combined zoom control

## Typography

All typography uses SwiftUI Dynamic Type text styles with the system font (San Francisco). No custom fonts.

| Role | SwiftUI Style | Design Variant | Weight | Usage |
|------|---------------|----------------|--------|-------|
| Display | `.largeTitle` | `.rounded` | `.bold` | Airport ICAO identifier in info sheet header |
| Heading | `.title3` | default | `.regular` | Airport name in info sheet, section headings |
| Heading (alt) | `.headline` | default | default | Flight category label, layer panel title |
| Body | `.subheadline` | default | `.medium` | Frequency values, weather data values, runway dimensions |
| Body (data) | `.subheadline` | `.monospaced` | `.medium` | Numeric data that must align (frequencies, wind direction, runway IDs) |
| Instrument value | `.title3` | `.monospaced` | `.semibold` | Instrument strip numeric readouts (GS, ALT, VS, TRK, DTG, ETE) |
| Label | `.caption` | default | `.medium` | Info chip labels, search placeholder, HUD distance/bearing |
| Label (data) | `.caption` | `.monospaced` | `.bold` | Nearest airport ICAO in HUD |
| Micro | `.caption2` | default | `.medium` / `.bold` | Instrument labels (GS, ALT, VS, TRK), section titles (RUNWAYS, FREQUENCIES, WEATHER), staleness badge text, unit labels |
| Raw data | `.caption2` | `.monospaced` | default | Raw METAR string |

**Typography rules:**
- Use `.monospaced` design variant for ALL numeric aviation data (speeds, altitudes, bearings, frequencies) to ensure digit alignment
- Use `.rounded` design variant ONLY for prominent identifiers (airport ICAO in sheet header)
- Two weights maximum in any single view: `.regular`/`.medium` for labels + `.semibold`/`.bold` for values
- Unit labels always `.caption2` and `.secondary` foreground, positioned adjacent to their value with `lastTextBaseline` alignment

## Color

This app uses SwiftUI semantic colors exclusively. No hardcoded hex values except for FAA-mandated flight category colors.

### Surface Colors (Material System)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | System background (map fills screen) | MapLibre base map covers 60%+ of viewport |
| Secondary (30%) | `.ultraThinMaterial` | Floating controls, instrument strip, nearest airport HUD, zoom buttons, mode toggle |
| Tertiary | `.regularMaterial` | Search bar, airport info sheet background (`.presentationBackground(.regularMaterial)`), layer toggle popover |

### Text Colors

| Role | Value | Usage |
|------|-------|-------|
| Primary text | `.primary` | Instrument values, airport ICAO, data values |
| Secondary text | `.secondary` | Labels, units, instrument labels, airport name, frequency type labels |

### Semantic Colors (FAA Standard -- Non-Negotiable)

| Color | SwiftUI Value | Hex Reference | Reserved For |
|-------|---------------|---------------|--------------|
| VFR | `.green` | #34C759 | Flight category dot, weather map dot, staleness badge (<30min) |
| MVFR | `.blue` | #007AFF | Flight category dot, weather map dot |
| IFR | `.red` | #FF3B30 | Flight category dot, weather map dot, staleness badge (>60min) |
| LIFR | `Color(red: 0.8, green: 0.0, blue: 0.8)` | #CC00CC | Flight category dot, weather map dot |

### Functional Colors

| Color | SwiftUI Value | Reserved For |
|-------|---------------|--------------|
| Danger/Alert | `.red` (solid background) | TFR disclaimer banner, NEAREST button background |
| Warning | `.orange` | Stale weather text, wind gusts, "No charts downloaded" banner |
| Aging | `.yellow` | Staleness badge 30-60min, runway lighting indicator |
| Informational | `.blue` | Bearing arrow in nearest HUD, route layer tint |
| Interactive | `.cyan` | Airport dots on map (GeoJSON circle layer) |
| Airspace | `.orange` | Airspace boundary tint on layer toggle |
| Navaids | `.purple` | Navaid layer tint |
| Stale | `.gray` | Weather data >2 hours old |

### Accent reserved for:
- Map mode toggle button tint (north-up / track-up)
- Tab bar selected tab indicator
- Active flight plan route line
- Search bar cursor color
- Toggle switch ON state (overridden per-layer with `tintColor`)

**Never use accent for:** Flight category colors, weather staleness, danger states, or data text.

## Animation Contracts

| Animation | Timing | Easing | Trigger |
|-----------|--------|--------|---------|
| Map mode toggle (north-up / track-up) | 0.2s | `.easeInOut` | User taps mode button |
| Initial map center to user GPS | 1.0s | `.easeInOut` (via MapLibre `setCenter(animated: true)`) | First `CLLocationUpdate` received after launch |
| Ownship position updates | Continuous | Linear interpolation (MapLibre `userTrackingMode`) | Every `CLLocationUpdate` (~1Hz) |
| Airport info sheet presentation | System default | System spring | User taps airport dot/annotation |
| Layer toggle popover | System default | System spring | User taps layer button |
| GPS "No Signal" indicator | 0.3s | `.easeIn` | GPS fix lost for >5 seconds |
| TFR banner appearance | 0.2s | `.easeInOut` | TFR layer toggled on |

**Animation rules:**
- Map animations always use MapLibre's native `animated: true` parameter, never SwiftUI `.animation()` on the map container
- Do NOT animate instrument strip value changes -- numeric data must update instantly for pilot situational awareness
- First GPS fix triggers ONE animated zoom from zoom 5 (CONUS) to zoom 10 (regional); subsequent GPS updates do NOT re-animate zoom
- Set `firstLocationReceived` flag after initial animation to prevent re-centering on every GPS update when user is panning

## Map Visual Contracts

### Ownship Indicator
- Shape: Aviation chevron/triangle (custom `MLNSymbolStyleLayer` with rotated icon)
- Color: `.blue` (system blue, matches ForeFlight/Garmin convention)
- Size: 32pt at all zoom levels (do not scale with map zoom)
- Rotation: Bound to `CLLocation.course` (degrees true)

### Airport Dots (GeoJSON Circle Layer)
- Default radius: 5pt
- Color: `.cyan` (system cyan) for all airports at high zoom
- Towered airports: Always visible at all zoom levels
- Non-towered airports: Hidden below zoom level 8, clustered between zoom 8-10, individual dots at zoom 10+
- Clustering zoom thresholds: cluster below zoom 10, de-cluster at zoom 10+

### Weather Dots (GeoJSON Circle Layer)
- Radius: 8pt (slightly larger than airport dots to be visible over them)
- Color: Data-driven by `flightCategory` GeoJSON property (VFR=green, MVFR=blue, IFR=red, LIFR=magenta)
- Border: 1pt white stroke for contrast against map

### Airspace Boundaries (GeoJSON Fill + Symbol Layers)
- Class B: Blue fill at 10% opacity, 2pt solid blue border
- Class C: Magenta fill at 10% opacity, 2pt solid magenta border
- Class D: Blue dashed fill at 8% opacity, 1.5pt dashed blue border
- Floor/ceiling labels: `.caption2` monospaced, positioned at polygon centroid

### TFR Polygons (GeoJSON Fill Layer)
- Fill: Red at 20% opacity
- Border: 2pt solid red
- Hatching: Diagonal line pattern at 45 degrees (if MapLibre supports), otherwise solid red fill at 15% opacity

### VFR Sectional Overlay
- Default opacity: 70% (0.7 `rasterOpacity`)
- Adjustable range: 0% to 100% via opacity slider
- Tile source: `mbtiles:///` URL scheme from downloaded MBTiles files

+------------------------------------------------------------------+
|  [TFR DISCLAIMER BANNER - full width, red bg, white text]        |  conditional

- Map fills entire screen, extends behind status bar (`.ignoresSafeArea(edges: .top)`)

- Top-left: Flight plan button + NEAREST button
- Top-right: Layer controls stack (mode, zoom, layers)
- Bottom: Stacked -- chart warning, nearest HUD, instrument strip
- Bottom spacing: 16pt horizontal padding, 8pt bottom padding from safe area

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Download Charts" (settings tab, chart region picker) |
| No GPS heading | "---" (dashes in instrument strip values) |
| No GPS indicator | "No GPS" (subtle text, not alarming) |
| No weather available | "No weather available" |
| Loading weather | "Loading weather..." (with spinner) |
| No charts downloaded | "No VFR charts downloaded" (orange capsule banner, tappable to settings) |
| TFR disclaimer | "TFR DATA IS SAMPLE ONLY -- NOT FOR NAVIGATION" (red banner, all caps, non-dismissable) |
| Stale weather (>2h) | "STALE" (gray badge, all caps) |
| Weather age | "<1 min" / "15 min" / "1h 30m" (relative time, progressive) |
| Search placeholder | "Search airports (ICAO, name, city)" |
| Layer panel title | "Map Layers" |
| Sample TFR note | "Sample data only" (red caption below TFR toggle) |
| Empty search | (no results shown, search clears) |
| Error state (weather fetch) | Silent failure -- search results clear, no error toast (weather is non-critical, map still works) |
| Error state (database) | App shows map without airport data -- no crash, graceful degradation |
| Error state (GPS denied) | Instrument strip shows "---", map shows CONUS view, app remains fully functional for chart viewing |

### Destructive Actions in Phase 1
None. Phase 1 has no delete, remove, or irreversible user actions. Chart downloads are additive. All map interactions are read-only.

## State Contracts

### GPS States

| State | Instrument Strip | Map | Nearest HUD |
|-------|-----------------|-----|-------------|
| Acquiring fix | "---" for all values | CONUS view (39N, 98W, zoom 5) | Hidden |
| Fix acquired | Live values updating ~1Hz | Animate to user position, zoom 10 | Shows nearest airport |
| Fix lost (>5s) | "---" + "No GPS" indicator | Last known position, no ownship movement | Last known nearest |
| Background | Values continue updating | N/A (app backgrounded) | N/A |

### Network States

| State | Weather dots | Airport info weather | Banner |
|-------|-------------|---------------------|--------|
| Online | Live data, 15min refresh | Auto-fetch on open | None |
| Offline | Cached data with staleness badge | Cached if available, "No weather available" if not | Reachability indicator (implementation detail for INFRA-02) |
| Transitioning | Last known data | Last known data | None (debounce transitions) |

### Chart States

| State | Map overlay | Indicator |
|-------|------------|-----------|
| No charts downloaded | No sectional overlay visible | Orange "No VFR charts downloaded" capsule (tappable) |
| Charts downloaded, current | Sectional overlay at user opacity | None |
| Charts expiring (<7 days) | Sectional overlay at user opacity | Phase 6 concern (expiration warning badge) |
| Charts expired | Sectional overlay at user opacity | Phase 6 concern (expired indicator) |
