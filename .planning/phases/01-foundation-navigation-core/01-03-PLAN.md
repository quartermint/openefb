    **LocationService.swift** â€” `final class LocationService: LocationServiceProtocol, @unchecked Sendable`:

    Uses `CLLocationUpdate.liveUpdates(.otherNavigation)` AsyncSequence (NOT CLLocationManager delegate). This is the iOS 17+ modern approach per RESEARCH.md.

    Properties:
    - `private var trackingTask: Task<Void, Never>?`
    - `private var backgroundSession: CLBackgroundActivitySession?`
    - `private(set) var isTracking: Bool = false`
    - `private weak var appState: AppState?`

    Methods:
    - `init(appState: AppState)` â€” Store weak reference to AppState
    - `func startTracking() async`:
      1. Create `CLBackgroundActivitySession()` and store it (enables background GPS per NAV-07)
      2. Set `isTracking = true`
      3. Create a `Task` that iterates `CLLocationUpdate.liveUpdates(.otherNavigation)`
      4. For each update where `update.location` is non-nil:
         - Extract speed (m/s to knots: `* 1.94384`), altitude (m to feet: `/ 0.3048`), course (degrees true)
         - Compute VSI from successive altitude samples: `(currentAlt - previousAlt) / timeDelta * 60` in feet per minute
         - Update AppState on MainActor:
           ```swift
           await MainActor.run {
               appState?.ownshipPosition = location
               appState?.groundSpeed = max(0, location.speed * 1.94384) // knots
               appState?.altitude = (location.altitude / 0.3048).rounded(toNearest: 10) // feet MSL, rounded to 10
               appState?.verticalSpeed = computedVSI // fpm
               appState?.track = location.course >= 0 ? location.course : 0 // degrees true
               appState?.gpsAvailable = true
           }
           ```
      5. Store the Task in `trackingTask`
    - `func stopTracking()`:
      1. `trackingTask?.cancel()`
      2. `backgroundSession?.invalidate()`
      3. Set `isTracking = false`

    Keep a `previousAltitude: Double?` and `previousAltitudeTime: Date?` for VSI computation. Reset on `startTracking()`.

    **ChartExpirationBadge.swift** â€” `struct ChartExpirationBadge: View` (INFRA-03):

    Displays chart cycle expiration status. Takes `daysRemaining: Int?` as input.
    - If `daysRemaining` is nil: no chart loaded, show nothing (handled by "No VFR charts downloaded" banner)
    - If `daysRemaining <= 0`: EXPIRED â€” red background, white text: "CHARTS EXPIRED" with `exclamationmark.triangle.fill` icon
    - If `daysRemaining > 0 && daysRemaining <= 7`: WARNING â€” yellow background, black text: "Charts expire in {N}d" with `exclamationmark.triangle.fill` icon
    - If `daysRemaining > 7`: CURRENT â€” no badge shown (charts are current)

    Style: capsule shape, `.caption2.semibold` font, 8pt horizontal / 4pt vertical padding. Positioned in LayerControlsView below the sectional opacity slider, and as an overlay on MapContainerView when charts are expired or expiring.

    ```swift
    struct ChartExpirationBadge: View {
        let daysRemaining: Int?

        var body: some View {
            if let days = daysRemaining, days <= 7 {
                HStack(spacing: 4) {
                    Image(systemName: "exclamationmark.triangle.fill")
                    Text(days <= 0 ? "CHARTS EXPIRED" : "Charts expire in \(days)d")
                }
                .font(.caption2)
                .fontWeight(.semibold)
                .padding(.horizontal, 8)
                .padding(.vertical, 4)
                .foregroundColor(days <= 0 ? .white : .black)
                .background(days <= 0 ? Color.red : Color.yellow)
                .clipShape(Capsule())
            }
        }
    }
    ```
