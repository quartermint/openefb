    Import FoundationModels. Create `@Observable @MainActor final class DebriefEngine`:

    1. **Published state:**
       - `private(set) var partialDebrief: FlightDebrief.PartiallyGenerated?` (streaming output for UI)
       - `private(set) var completedDebrief: FlightDebrief?` (final result after stream completes)
       - `private(set) var isGenerating: Bool = false`
       - `private(set) var error: Error?`
       - `private(set) var availabilityStatus: AvailabilityStatus = .unknown`

    2. **Availability enum (must be testable independently of Foundation Models runtime):**
       ```swift
       enum AvailabilityStatus: Equatable, Sendable {
           case unknown
           case available
           case unavailable(reason: String)

           /// Maps a Foundation Models unavailable reason to a user-friendly string.
           /// Extracted as a static method so it can be unit tested without a real device.
           static func reasonMessage(for reasonDescription: String) -> String {
               if reasonDescription.contains("deviceNotEligible") || reasonDescription.contains("not eligible") {
                   return "This device does not support Apple Intelligence."
               } else if reasonDescription.contains("appleIntelligenceNotEnabled") || reasonDescription.contains("not enabled") {
                   return "Apple Intelligence is not enabled. You can enable it in Settings."
               } else if reasonDescription.contains("modelNotReady") || reasonDescription.contains("not ready") {
                   return "Apple Intelligence is still setting up. Please try again later."
               } else {
                   return "AI Debrief is currently unavailable."
               }
           }
       }
       ```

    3. **Session management:**
       - `private var session: LanguageModelSession?`

    4. **checkAvailability() method:**
       - Check `SystemLanguageModel.default.availability`
       - Switch on result:
         - `.available`: set `availabilityStatus = .available`
         - `.unavailable(let reason)`: set `availabilityStatus = .unavailable(reason:)` where reason string is derived via `AvailabilityStatus.reasonMessage(for:)`. Map `.deviceNotEligible` to "This device does not support Apple Intelligence.", `.appleIntelligenceNotEnabled` to "Apple Intelligence is not enabled. You can enable it in Settings.", `.modelNotReady` to "Apple Intelligence is still setting up. Please try again later."

    5. **prewarm(systemPrompt: String) method:**
       - Guard availabilityStatus == .available
       - Create `session = LanguageModelSession(instructions: systemPrompt)`
       - Call `session?.prewarm()`

    6. **generateDebrief(prompt: String, recordingDB: RecordingDatabase, flightID: UUID) async method:**
       - Guard session is not nil, set isGenerating = true, clear error and partialDebrief
       - Defer { isGenerating = false }
       - System prompt (instructions for session): "You are an experienced flight instructor reviewing a student pilot's VFR flight. Provide constructive, specific feedback based on the flight data. Focus on safety, decision-making, and technique. Be encouraging but honest about areas for improvement."
       - Create session if nil: `session = LanguageModelSession(instructions: systemPrompt)`
       - `let stream = session!.streamResponse(to: prompt, generating: FlightDebrief.self)`
       - `for try await partial in stream { self.partialDebrief = partial.content }`
       - After stream completes: extract the final FlightDebrief from the completed response
       - Set `completedDebrief = finalDebrief`
       - Create DebriefRecord via `DebriefRecord.fromFlightDebrief(finalDebrief, flightID: flightID, promptTokensEstimate: nil, responseTokensEstimate: nil)`
       - Call `try recordingDB.insertDebrief(record)` to persist (INSERT OR REPLACE)
       - Catch errors: set `self.error = error`

    7. **loadExistingDebrief(recordingDB: RecordingDatabase, flightID: UUID) method:**
       - Try `recordingDB.debrief(forFlight: flightID)`
       - If found, reconstruct FlightDebrief from DebriefRecord fields and set completedDebrief

    8. **discard() method:**
       - Set session = nil, partialDebrief = nil, completedDebrief = nil, error = nil

    NOTE: The `streamResponse` API returns `AsyncThrowingStream` where each element has a `.content` property typed as `FlightDebrief.PartiallyGenerated`. After the stream completes successfully, the final element's content can be used. However, to get the fully typed `FlightDebrief`, use `session.respond(to:generating:)` for the non-streaming path, or collect the final streamed result. The Research doc shows the pattern. If `streamResponse` does not directly yield the final typed object, use the last partial and reconstruct, OR call `respond` separately after streaming for persistence. Prefer: keep streaming for UI, then after stream ends create DebriefRecord from the partial's fields (narrativeSummary, phaseObservations, improvements, overallRating may all be non-nil at stream end).
