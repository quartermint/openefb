    **7. Create FlightPhaseDetector (efb-212/Services/Recording/FlightPhaseDetector.swift):**
    A struct (not actor) -- pure function state machine consuming TrackPointRecord values.

    Per user decision: speed + altitude state machine with hysteresis. Thresholds from SFR:
    - preflight: groundSpeed < 5 kts
    - taxi: 5 <= groundSpeed < 15 kts (or > 15 but not climbing)
    - takeoff: groundSpeed > 15 kts AND verticalSpeed > 200 fpm (climbing)
    - departure: groundSpeed > 60 kts AND altitude > 500 ft AGL AND still climbing
    - cruise: altitude > 500 ft AGL AND |verticalSpeed| < 200 fpm (level flight)
    - approach: verticalSpeed < -200 fpm (descending) AND altitude < previousCruiseAltitude
    - landing: groundSpeed < 15 kts after a descent phase
    - postflight: groundSpeed < 5 kts sustained after landing

    Uses 5-point rolling average for smoothing. 30-second minimum hysteresis per user decision. `mutating func process(_ point: TrackPointRecord) -> FlightPhaseType`. Returns the current phase after evaluating.

    Stores `recentPoints: [TrackPointRecord]` (max 5), `currentPhase`, `phaseEnteredAt: Date`, `hysteresisSeconds: TimeInterval = 30`.
