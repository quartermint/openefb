    **1. Create efb-212/Core/Extensions/CLLocationCoordinate2D+GreatCircle.swift:**

    Extension on `CLLocationCoordinate2D` with a static function:
    ```swift
    static func greatCirclePoints(
        from start: CLLocationCoordinate2D,
        to end: CLLocationCoordinate2D,
        count: Int = 100
    ) -> [CLLocationCoordinate2D]
    ```

    Implementation uses spherical interpolation (slerp) as documented in 02-RESEARCH.md:
    - Convert lat/lon to radians using existing `.degreesToRadians` extension on Double
    - Compute angular distance `d = 2 * asin(sqrt(pow(sin((lat1 - lat2) / 2), 2) + cos(lat1) * cos(lat2) * pow(sin((lon1 - lon2) / 2), 2)))`
    - Guard `d > 0` returning `[start]` for zero-distance
    - Map over `0...count` computing intermediate point at fraction `f = Double(i) / Double(count)`:
      - `a = sin((1 - f) * d) / sin(d)`
      - `b = sin(f * d) / sin(d)`
      - `x = a * cos(lat1) * cos(lon1) + b * cos(lat2) * cos(lon2)`
      - `y = a * cos(lat1) * sin(lon1) + b * cos(lat2) * sin(lon2)`
      - `z = a * sin(lat1) + b * sin(lat2)`
      - `lat = atan2(z, sqrt(x * x + y * y))`
      - `lon = atan2(y, x)`
    - Convert back to degrees using `.radiansToDegrees`
    - Return array of CLLocationCoordinate2D

    Mark function as `nonisolated` if needed for Sendable compliance.

    **Database service injection (resolving the shared service dependency):**

    FlightPlanViewModel takes `databaseService: any DatabaseServiceProtocol` in init. The AviationDatabase is created in MapContainerView.onAppear and shared via `appState.sharedDatabaseService`. However, if the user navigates to the Flights tab before the Map tab has loaded, `appState.sharedDatabaseService` will be nil.

    **Concrete fallback strategy:** In `.onAppear`, if `appState.sharedDatabaseService` is nil, initialize AviationDatabase directly:
    ```swift
    .onAppear {
        guard viewModel == nil else { return }
        let dbService: any DatabaseServiceProtocol
        if let shared = appState.sharedDatabaseService {
            dbService = shared
        } else {
            // Initialize AviationDatabase directly when Map tab hasn't loaded yet.
            // AviationDatabase opens the bundled SQLite read-only -- safe to have multiple instances.
            let aviationDB = AviationDatabase()
            try? aviationDB.openBundledDatabase()
            dbService = aviationDB
            // Also set as shared so other tabs can use it
            appState.sharedDatabaseService = aviationDB
        }
        let vm = FlightPlanViewModel(databaseService: dbService, appState: appState)
        vm.configure(mapService: appState.sharedMapService, modelContext: modelContext)
        viewModel = vm
    }
    ```

    Note: `AviationDatabase.openBundledDatabase()` opens the bundled SQLite file read-only. Creating a second instance is safe (SQLite WAL mode supports concurrent readers). The `try?` is acceptable because search will gracefully show empty results if the database fails to open.

    Do NOT use a `PlaceholderDatabaseService` -- use the real AviationDatabase as fallback.
